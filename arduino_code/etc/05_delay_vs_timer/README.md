# delay (동기) vs millis (비동기) 비교 예제

## 📌 개요
`delay()` 방식과 `millis()` 방식의 차이를 비교하는 예제입니다. 비동기 프로그래밍의 중요성을 이해할 수 있습니다.

## 🔧 하드웨어
- **LED** 3개 (서로 다른 주기로 깜빡임)
- **버튼** 1개 (반응 속도 테스트)

## 📍 핀 연결
```
Arduino UNO
├─ D3  → LED 1 (500ms 주기)
├─ D4  → LED 2 (1000ms 주기)
├─ D5  → LED 3 (1500ms 주기)
└─ D2  → 버튼 (풀업)
```

## 🎯 비교 내용

### delay() 방식 (동기)
```cpp
digitalWrite(LED1, HIGH);
delay(500);  // ← 이 시간 동안 아무것도 할 수 없음!
digitalWrite(LED1, LOW);
delay(500);
```

**문제점**:
- ❌ delay 중에는 다른 작업 불가
- ❌ 버튼 입력 무시 (반응 느림)
- ❌ 여러 작업 동시 처리 불가
- ❌ 코드 확장성 낮음

### millis() 방식 (비동기)
```cpp
unsigned long currentMillis = millis();

if (currentMillis - previousMillis >= interval) {
  previousMillis = currentMillis;
  // LED 토글
}

// 버튼은 항상 체크 가능!
```

**장점**:
- ✅ 여러 작업 동시 처리
- ✅ 버튼 즉시 반응
- ✅ 정밀한 타이밍 제어
- ✅ 코드 확장성 높음 (권장!)

## 💻 Serial 명령어
| 명령어 | 기능 |
|--------|------|
| `millis` | millis() 방식 (비동기, 권장) |
| `delay` | delay() 방식 (동기, 비교용) |

## 📊 테스트 방법
1. 아두이노에 업로드
2. `millis` 모드에서 버튼 눌러보기 → **즉시 반응!**
3. `delay` 모드로 전환
4. 버튼 눌러보기 → **반응 느림!**

## 📝 성능 비교

| 항목 | delay() | millis() |
|------|---------|----------|
| 버튼 반응 속도 | 느림 (최대 6초 지연) | 즉시 (< 10ms) |
| 여러 작업 동시 처리 | 불가능 | 가능 |
| CPU 효율 | 낮음 | 높음 |
| 코드 확장성 | 낮음 | 높음 |
| 권장 여부 | ❌ | ✅ |

## 🔍 응용 분야
- 실시간 센서 모니터링
- 다중 타이머 시스템
- 비차단 지연 (Non-blocking delay)
- 복잡한 자동화 시스템

## 💡 핵심 개념
**비동기 프로그래밍**은 여러 작업을 동시에 처리할 수 있게 해주는 중요한 기법입니다. 실제 프로젝트에서는 항상 `millis()` 방식을 사용하는 것이 좋습니다!
