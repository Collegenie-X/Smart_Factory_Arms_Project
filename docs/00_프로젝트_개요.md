# 스마트 팩토리와 스마트 팔 프로젝트 수업 개요

## 프로젝트 목표

**인간이 잘하는 것에 집중하라!**
- 진짜 문제 발견하기
- 깊이 있는 성찰하기
- 창의적인 비즈니스 모델 만들기

이 프로젝트는 학생들이 스마트 팩토리와 로봇 팔 기술을 활용하여 **실제 산업 현장의 문제**를 발견하고, 
**피지컬 AI 기술**로 해결책을 제시하며, **공모전 수준의 결과물**을 만드는 것을 목표로 합니다.

## 핵심 학습 철학: 바이브 코딩 (VIBE Coding)

바이브 코딩은 4가지 역할을 순환하며 프로젝트를 완성하는 방법론입니다:

### 1. 기획자 (Visionary) - 진짜 문제 찾기
- **인간의 강점**: 공감, 관찰, 맥락 이해
- **활동**: 현장 인터뷰, 문제 정의, 사용자 니즈 파악
- **질문**: "누가 어떤 상황에서 왜 불편한가?"

### 2. 실행자 (Implementer) - 만들기
- **기술**: 아두이노, 로봇 팔, 스마트폰 AI, 컴퓨터 비전
- **활동**: 설계, 코딩, 조립, 테스트
- **질문**: "어떻게 작동하게 만들 것인가?"

### 3. 디버거 (Debugger) - 성찰하기
- **인간의 강점**: 비판적 사고, 원인 분석, 개선
- **활동**: 오류 분석, 실패 원인 탐구, 개선 방향 도출
- **질문**: "왜 실패했는가? 무엇을 배웠는가?"

### 4. 기업가 (Entrepreneur) - 비즈니스 만들기
- **인간의 강점**: 가치 창출, 시장 이해, 지속가능성 설계
- **활동**: 비즈니스 모델 설계, 시장 조사, 수익 구조 기획
- **질문**: "누구에게 어떤 가치를 제공하여 어떻게 돈을 벌 것인가?"

## 기술 구성

### 하드웨어
- **교구재 1**: [ICBANQ P013154977](https://www.icbanq.com/P013154977) - 로봇 팔 메커니즘
- **교구재 2**: [ICBANQ P014847535](https://www.icbanq.com/P014847535) - 스마트 팩토리 모듈
- **아두이노**: 센서 제어 및 액추에이터 구동
- **센서류**: 힘 센서, 거리 센서, 온도 센서 등

### 소프트웨어 & AI
- **스마트폰 카메라**: 비전 센서 역할 (카메라가 없는 아두이노의 한계 극복)
- **컴퓨터 비전**: 
  - OpenCV (객체 인식)
  - MediaPipe (손 추적, 포즈 인식)
  - TensorFlow Lite (경량 AI 모델)
- **통신**: Bluetooth, Wi-Fi (아두이노 ↔ 스마트폰)

### 피지컬 AI 학습 방법
```
현실 문제 → 데이터 수집 → 모델 학습 → 시뮬레이션 테스트 → 실제 적용 → 피드백 → 개선
```

## 학습 대상
- **중등 ~ 고등학생** 또는 대학 예비 단계
- 피지컬 컴퓨팅에 관심 있는 학습자
- 공모전 참가를 목표로 하는 팀 (3-4명 권장)

## 수업 시간
- **총 28차시** (차시당 45-50분 기준)
- 약 7주 과정 (주 4차시 진행 시)

## 최종 결과물
1. **작동하는 프로토타입**: 로봇 팔 + 스마트폰 AI 통합 시스템
2. **문제 해결 보고서**: 진짜 문제 정의 → 해결 과정 → 성찰
3. **비즈니스 모델 캔버스**: 시장성 있는 사업 계획
4. **발표 자료 및 시연**: 공모전 제출용 포트폴리오

## 평가 기준
| 영역 | 비중 | 세부 내용 |
|------|------|-----------|
| 문제 발견 | 25% | 문제의 진정성, 명확성, 해결 가치 |
| 기술 구현 | 25% | 작동 완성도, 기술적 난이도, 창의성 |
| 성찰 및 개선 | 20% | 실패 분석, 학습 내용, 개선 과정 |
| 비즈니스 모델 | 20% | 시장성, 수익 구조, 실현 가능성 |
| 발표 및 문서화 | 10% | 설득력, 완성도, 협업 |

## 학습 철학

### 인간 중심 AI 교육
```
AI가 할 수 있는 것: 패턴 인식, 반복 작업, 빠른 계산
인간이 잘하는 것: 문제 정의, 공감, 창의성, 윤리적 판단, 비즈니스 설계
```

### 실패를 통한 학습
- 시뮬레이션과 현실의 차이 (Reality Gap) 경험
- 센서 오류, 환경 변화, 예측 불가능성 극복
- "왜 실패했는가?"를 분석하는 능력 = 진짜 엔지니어의 역량

### 지속 가능한 기술
- 환경: 전력 소비, 재료 선택, 폐기 방법
- 경제: 비용 대비 효과, 유지보수 비용
- 사회: 일자리 변화, 안전, 윤리
